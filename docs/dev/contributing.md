# Contributing

## Contribution Rules

1. Do not introduce new behavior without tests.
2. Keep APIs small and explicit.
3. Prefer readability and correctness over optimization.
4. Add or update docs for any contract change.

## Pull Request Checklist

1. Relevant tests added or updated.
2. Architecture docs updated if contracts changed.
3. Lint and test commands pass.
4. Scope is focused and reviewable.

## Code Review Focus

1. Contract correctness
2. Error handling quality
3. Determinism of behavior
4. Test clarity and coverage rationale
